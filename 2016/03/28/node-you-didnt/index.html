Node you didn't

28 March 2016

Asynch(irony)

A core node module built to handle file i/o, fs, has been the focus of my learning curve these past couple of hours. Up until this point in my life, I had only dealt with sending raw, server-generated data via http using node's core http module.

Now I am faced with serving static assets to a client. After sending uninspiring url strings to the clients, showing off the locations of the assets the clients were missing out on, my pair partner and I found an answer: a client needed to be hand-fed data.

Mental Debugging

Time to debug my mental model of the client-server relationship… with a RESTful web service, the client cannot directly access any resources hosted by the server that are not explicitly prepared by the server itself.

To tie it all up, readFile is an fs module method we discovered that will parse a file into a data structure that is ready to send on the response object of an HTTP request.

Note: Handling the HTTP response.end method beyond the scope of an asynchronous routine will guarantee that the response object will not wait on the completion of the asynchronous routine. A readFile method is such an example.
