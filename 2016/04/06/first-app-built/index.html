Scope

I built a project snippet utilizing several open-sourced APIs, the result was a weather-matching service to match geographic regions based off of real-time weather conditions.

Here is the app and the corresponding source code. Have fun.

Takeaways

APIs

After playing with the first API (OpenWeatherMap), I decided to integrate a forward geocoding API (OpenCageGeocoding) to extend the data querying scope from the initial API. To increase accuracy and performance, I should setup a cluster of servers to handle the data streams independently.

Stack

I used AngularJS to handle the front-end and ExpressJS to handle the data channels. To improve on the real-time aspect of this service, I could use a network of CDNs to cache segments of incoming weather data, reducing data retrieval delays and focusing computational efforts regionally.
